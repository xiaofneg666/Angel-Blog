### 问题总结
本次遇到的问题是： 克隆仓库后直接运行开发脚本，因缺少项目依赖导致命令执行失败 。具体表现为：

1. 前端运行 npm run dev 时，报错 'vite' 不是内部或外部命令
2. 后端运行 npm run dev 时，报错 'nodemon' 不是内部或外部命令
### 根本原因
- 项目依赖（如 vite 、 nodemon ）未安装，这些命令需要通过 npm install 下载后才能使用
- 该项目是 前后端分离架构 ，前端和后端有各自的 package.json 文件，需要分别安装依赖
### 下次遇到类似问题的解决步骤
1. 克隆仓库后，先检查项目结构 ：确认是否为前后端分离项目（查看是否有 client 、 server 等子目录）
2. 进入对应目录，安装依赖 ：
   - 前端：进入 client 目录，运行 npm install
   - 后端：进入 server 目录，运行 npm install
3. 依赖安装完成后，再运行开发脚本 ：
   - 前端： npm run dev （启动 Vite 开发服务器）
   - 后端： npm run dev （启动 nodemon 开发服务器）
4. 检查运行状态 ：确认服务器是否成功启动，查看终端输出的访问地址和状态信息
### 关键注意事项
- npm install 会安装 package.json 中所有依赖（包括 dependencies 和 devDependencies ）
- 开发依赖（如 nodemon 、 vite ）通常用于开发环境，通过 npm install 可以自动安装
- 前后端分离项目需要 分别安装依赖并启动服务 ，才能实现完整功能
通过以上步骤，可以确保项目依赖完整，开发服务器能够正常启动运行。
